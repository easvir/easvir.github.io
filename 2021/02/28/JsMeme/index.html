这三个要合起来聊，这里就得提到这篇文章 What is {} + {} in JavaScript?
在第一节中提到：

The plus operator performs three kinds of conversion: It converts values to primitives, numbers and strings.

加法运算符的作用是数值求和，或者字符串拼接，而JS又不像C++一样可以进行运算符重载，因此在进行对象之间的加法时就要将对象进行转换，第一步就是调用ToPrimitive()尝试将对象转换为基本类型。

The following steps are performed to convert a value input (§9.1):

  1. If input is primitive, return it as is.
  2. Otherwise, input is an object. Call obj.valueOf(). If the result is primitive, return it.
  3. Otherwise, call obj.toString(). If the result is a primitive, return it.
  4. Otherwise, throw a TypeError.

当获取到基本类型后，就尝试将其转换为字符串或数字。


由此可以分析789的转换过程了。

7、

Converting [] to a primitive first tries valueOf() which returns the array itself (this):

> var arr = [];
> arr.valueOf() === arr
true

As that result is not a primitive, toString() is called next and returns the empty string (which is a primitive). Therefore, the result of [] + [] is the concatenation of two empty strings.

首先调用valueOf()，空数组和空对象的返回值都是其本身，于是下一步转换为字符串，空数组返回空字符串，两者拼接，返回空字符串。

8、空对象转换为字符串是'[object Object]'，与空字符串拼接返回'[object Object]'。

9、

This is interpreted as a code block followed by +[]. The following expressions are equivalent:

+[]
Number([])
Number([].toString()) // [].valueOf() isn’t primitive
Number(“”)
0

这里将空对象解释为空的代码块跳过执行，返回的内容为+[]，将空字符串转换为数字，返回0。

在这里8和9是不同的，当添加括号后({}+[])和([]+{})返回值一致，都是'[object Object]'，原因就是没有括号时导致前置的花括号被解释为空代码块。

此处还有一个字符串的转换要分析，放到后面一块讲。

13、 true===1

MDN Link

JavaScript提供三种不同的值比较操作：
-严格相等比较 (也被称作”strict equality”, “identity”, “triple equals”)，使用 === ,
-抽象相等比较 (“loose equality”，”double equals”) ，使用 ==
-以及 Object.is （ECMAScript 2015/ ES6 新特性）

==比===多了一步可能的类型转换。

在 ES2015 以前，你可能会说双等和三等是“扩展”的关系。比如有人会说双等是三等的扩展版，因为他处理三等所做的，还做了类型转换。例如 6 == “6” 。反之另一些人可能会说三等是双等的扩展，因为他还要求两个参数的类型相同，所以增加了更多的限制。怎样理解取决于你怎样看待这个问题。

15、 9+”1”

ECMA Link
这里直接参考ECMA规范。

  1. If Type(lprim) is String or Type(rprim) is String, then:
    a.Let lstr be ToString(lprim).
    b.ReturnIfAbrupt(lstr).
    c.Let rstr be ToString(rprim).
    d.ReturnIfAbrupt(rstr).
    e.Return the String that is the result of concatenating lstr and rstr.

由此可以看出，在加法中，字符串拼接拥有更高的优先级，这也说明为什么[]+[]的结果是""而不是0。
